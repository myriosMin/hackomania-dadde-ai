Create a high-fidelity desktop main website landing page in Figma for a community-powered disaster relief platform. This platform allows users to discover urgent disaster campaigns, donate directly, and eventually access personalised giving features after logging in.

The design should feel modern, trustworthy, humanitarian, transparent, and slightly fintech-inspired. It should balance emotional urgency with clarity, safety, and credibility. The page should look like a real startup product homepage/dashboard hybrid, not a charity poster. Use a clean grid layout, soft rounded cards, subtle shadows, accessible spacing, and strong visual hierarchy.

Scope

Design only the main website main page.

Do not design:

the vendor checkout flow

external merchant pages

the logged-in user dashboard page in full

the onboarding flow pages in full

You may visually hint at those flows where relevant, but the actual deliverable is just the main page of the website.

Main Purpose of This Page

This page is the first screen users see when they visit the domain. Its role is to:

introduce the platform

show urgent or recommended disaster relief campaigns

encourage donation

guide users toward signing up or logging in

hint that logged-in users get access to more personalised donation tracking features

Page Structure

Create a polished desktop layout with these sections:

Top navigation bar

Hero / introduction section

Main disaster campaign dashboard section

Optional trust / transparency metrics section

Footer

1. Navigation Bar

Include a top navigation bar with:

platform logo on the left

“Main Page”

a second page item, such as “My Giving” or “Dashboard”

“Profile”

“Settings”

Important behavior to reflect

This project includes a second page that is only fully accessible to logged-in users.

For this main page design, represent that clearly:

the Main Page is active

the second page should appear present in the nav, but restricted for users who are not logged in

this restricted state can be shown with:

a lock icon

muted styling

small label like “Login required”

The Profile button should visually imply that it leads to the log in / sign up flow.

2. Hero / Introduction Section

Below the nav bar, create a hero section that explains the platform in a clear and compelling way.

Include:

a strong headline about helping communities respond to disasters faster

a short supporting paragraph explaining that users can contribute through direct donations and monthly pledges

a primary CTA button such as:

“Support a Disaster”

“Start Donating”

a secondary CTA button such as:

“Learn How It Works”

“View Transparency”

Optional right-side visual:

a world map with highlighted disaster zones

a humanitarian/relief-themed illustration

or a sleek live-fund summary card

The hero should make the platform feel credible, impactful, and easy to understand.

3. Main Disaster Campaign Dashboard Section

This is the core of the page.

Create a central dashboard section showing a curated selection of recommended or urgent disaster relief campaigns, prioritised by the platform’s recommendation algorithm.

At the top of this section, include:

a section title such as:

“Urgent & Recommended Relief Campaigns”

“Disasters Needing Support Right Now”

a short explanatory sentence saying these campaigns are prioritised by urgency, relevance, and impact

Disaster Campaign Cards

Display the campaigns in a clean card grid.

Each card should include:

an image of the disaster

the disaster name

a short description

a donation button at the bottom

Optional supporting metadata on each card:

location

disaster category tag

urgency label

affected population

funding progress

Example campaigns:

Flood Relief in the Philippines

Earthquake Response in Turkey

Wildfire Recovery in California

Typhoon Support in Southeast Asia

Donate Button Behavior

Reflect both user states in the UI design:

Logged-in user state

When the user clicks the donation button:

it changes to a “Selected” state

show this as a clear active style:

filled button

checkmark icon

text changed to “Selected”

Not logged-in user state

When the user clicks the donation button:

they should be redirected to a log in / sign up pop-up

To reflect this in the mockup, include either:

one sample modal overlay for login/sign-up

or one highlighted interaction state showing that clicking donate triggers authentication

The pop-up should look simple and modern, with:

log in option

sign up option

4. Profile / Sign-Up Flow Context

The Profile button leads users into the authentication flow.

During sign-up, users will:

enter basic personal information

link their Open Payments account

After sign-up, users go through onboarding:

Donation Preferences Page

Monthly Pledge Page

The monthly pledge page includes:

an amount input field

a Skip button for users who do not want to pledge yet

You do not need to design these pages here, but the main page should feel consistent with this product ecosystem.

5. Logged-In-Only Second Page Context

There is a second page in the platform that is only fully accessible to logged-in users.

That page contains:

how much the user has donated this month

donation history

a Pledge button that redirects to the monthly pledge page

For this main page design, do not build that full page yet.

Only hint at it by:

showing it in the nav as locked or login-gated

optionally including a blurred preview card or locked panel teaser somewhere on the page

If shown, the locked state should feel elegant and product-like, not dramatic.

6. Transparency / Trust Section

Below the disaster campaign section, include a trust-building section that reinforces platform legitimacy and transparency.

Possible UI elements:

total community funds available

active disaster campaigns

total donations distributed

number of contributors

short copy about transparent fund tracking and responsible payout rules

This section should make the platform feel accountable, data-driven, and safe for donors.

7. Footer

Include a clean footer with items such as:

About

How It Works

Transparency

FAQ

Contact

Terms

Privacy

Open Payments

Visual Style Direction

Use a style that is:

modern SaaS meets social impact

clean and minimal

warm but credible

not childish

not overly corporate

accessible and polished

Suggested design traits:

soft neutral or slightly off-white background

one strong accent colour for CTA buttons

real disaster imagery in cards

subtle shadows

rounded corners

elegant iconography

clean sans-serif typography

balanced whitespace

Deliverable Requirement

Generate a Figma-ready desktop mockup of the main website page only.

The page should be complete enough to serve as the basis for an MVP homepage/dashboard.

It should clearly communicate:

what the platform does

what users can support

how users donate

the difference between logged-in and non-logged-in behavior

that deeper personalised features exist behind login

Do not generate:

vendor site screens

merchant checkout flows

the full logged-in dashboard page

onboarding pages in full

Only create the main website main page.
