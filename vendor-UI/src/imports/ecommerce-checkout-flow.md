Create a high-fidelity desktop e-commerce website flow in Figma for a generic online store with random everyday consumer products. The purpose of this design is to simulate a normal merchant website that integrates a disaster-relief round-up donation option powered by DADDE.

Overall style:
- clean, modern, generic e-commerce design
- realistic but brand-neutral
- trustworthy, simple, polished
- similar to a standard online retail website
- not luxury, not overly trendy, not niche
- neutral product categories and simple UI
- desktop-first layout

Design 2 connected pages:
1. Generic e-commerce product page / storefront
2. Checkout payment page

PAGE 1 — GENERIC E-COMMERCE PRODUCT PAGE

Create a normal online shopping page with:
- top navigation bar
- store logo
- search bar
- product category links
- shopping cart icon
- account icon
- hero banner or featured products section
- grid of random products

Use random generic products such as:
- headphones
- water bottle
- keyboard
- tote bag
- lamp
- sneakers
- skincare item
- notebook
- phone case
- hoodie

Each product card should include:
- product image
- product name
- short product description
- price
- rating or review count
- “Buy Now” button
- optional “Add to Cart” button

The site should look like a believable general marketplace or small online store.

Important:
- the “Buy Now” button is the key CTA
- visually imply that clicking “Buy Now” leads to the payment page

PAGE 2 — CHECKOUT / PAYMENT PAGE

Design a standard checkout payment page for one selected product.

Layout should include:
- breadcrumb or checkout progress indicator
- product summary card
- item image
- item name
- quantity
- subtotal
- shipping fee
- total amount
- payment method section
- confirm payment button

Include standard payment methods such as:
- credit / debit card
- PayPal or digital wallet
- bank transfer
- Open Payments

Most important:
The Open Payments option should be clearly visible and feel like a featured or innovative payment option.

OPEN PAYMENTS INTERACTION

When the user selects “Open Payments”, the UI should expand below it to reveal a donation round-up module.

This expanded module should include:
- a short explanation that the user can round up their purchase to the nearest 10th cent
- clear wording that the extra amount automatically contributes to disaster response
- mention that this contribution is powered and managed by DADDE AI
- a toggle, checkbox, or selectable option to enable the round-up
- a small example amount breakdown

Example:
- Product price: $24.73
- Round up to: $24.80
- Contribution: $0.07

The round-up section should feel:
- easy to understand
- optional, not forced
- seamlessly integrated into checkout
- trustworthy and lightweight

Include supporting text such as:
“Round up your purchase to the nearest 10th cent and automatically contribute the difference to disaster response, powered and managed by DADDE AI.”

Also include:
- a small text link or button that says something like:
  - “Learn more about DADDE”
  - “Visit DADDE”
  - “About the disaster response fund”
- this should link conceptually to the main DADDE website

DADDE BRAND REFERENCE

The vendor page should still feel like the merchant’s website first, not the DADDE website.

DADDE should appear only in:
- the Open Payments expanded section
- the donation explanation
- the small external link back to the DADDE main website

Do not make DADDE the dominant brand on the vendor page.
It should feel like a third-party social impact payment feature embedded into a standard e-commerce checkout.

VISUAL DETAILS

Use:
- clean cards
- modern form fields
- realistic buttons
- neutral typography
- accessible spacing
- subtle borders and shadows
- believable checkout interaction patterns

For the Open Payments section, use:
- a slightly highlighted card or bordered section
- a trust badge or icon
- small note that this is secure and transparent

DELIVERABLE

Generate:
- one desktop generic e-commerce storefront page
- one desktop payment / checkout page
- clear Open Payments selection state
- expanded round-up donation UI
- subtle link back to the main DADDE website

The final design should clearly communicate this flow:
generic product browsing → Buy Now → payment page → select Open Payments → expand round-up donation option → optional link to DADDE website
